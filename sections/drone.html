🚁 Trilha — IA para Drones e Sistemas Aéreos
Do zero ao engenheiro de sistemas UAS com IA · ~18 meses · Fusão de controle de voo, visão e navegação autônoma
Drones com IA estão revolucionando agronegócio, mapeamento, inspeção de infraestrutura, logística e defesa. No Brasil, o mercado de agro-drones cresce 40%+ ao ano — oportunidade enorme para quem está em Goiás.

Ecossistema e mercado

Aplicações com maior demanda

  • Agronegócio: pulverização de precisão, mapeamento de lavoura, estimativa de NDVI, detecção de pragas
  • Inspeção: linhas de transmissão, oleodutos, pontes, torres eólicas
  • Mapeamento: fotogrametria, LiDAR aéreo, modelos 3D de terreno
  • Entrega de carga: last-mile delivery (Amazon, Wing)
  • Busca e resgate: drones com câmera térmica
  • Defesa: ISR (inteligência, vigilância, reconhecimento)

Fase 1 — Fundação de voo e controle (Meses 1–3)

Mês 1 — Princípios de voo e hardware
  • Aerodinâmica básica: sustentação, arrasto, empuxo, peso
  • Tipos de drone: multirotor, asa fixa, VTOL — quando usar cada um
  • Componentes: ESC, motores brushless, FC (flight controller), GPS, IMU, RC
  • Bateria LiPo: cuidados, C-rating, cálculo de autonomia
  • Regulatório: ANAC RBAC-E 94, classes de drone, habilitação
Recurso: ArduPilot docs + canal Painless360 (YouTube) — melhor canal técnico de drone em inglês
Mês 2 — Flight Controller e PX4/ArduPilot
  • Instalação e configuração PX4 ou ArduPilot (ambos open-source)
  • MAVLink: protocolo de comunicação GCS ↔ drone
  • QGroundControl / Mission Planner: planejamento de missão
  • Modos de voo: manual, stabilize, loiter, auto, guided
  • Simulação: SITL (Software In The Loop) — voar sem hardware
Mês 3 — Controle programático com Python
  • MAVSDK-Python: controlar drone via código Python
  • DroneKit: missões autônomas programáticas
  • Comandar decolagem, navegação para waypoints, pouso
  • Telemetria: ler altitude, velocidade, bateria em tempo real
  • Projeto: missão autônoma simulada no SITL + lógica em Python

Fase 2 — IA embarcada e percepção (Meses 4–8)

Mês 4 — Computer Vision no drone
  • Integrar câmera com pipeline de visão (OpenCV + PyTorch)
  • Detecção de objetos leve: YOLO-nano, MobileNet para hardware limitado
  • Gimbal e estabilização de câmera para qualidade de imagem
  • Streaming de vídeo para processamento em ground station
  • Projeto: drone detectando alvos em simulação (AirSim ou Gazebo)
Recurso: Microsoft AirSim (simulador realístico com Python API)
Mês 5 — Fotogrametria e mapeamento
  • Planejamento de voo para mapeamento: GSD, overlap, altitude
  • OpenDroneMap (ODM): reconstrução 3D de imagens aéreas
  • Ortomosaicos: geração e análise
  • NDVI: índice de vegetação a partir de câmera multiespectral
  • QGIS: análise espacial dos resultados
  • Projeto: mapeamento de área e análise de vegetação
Mês 6–7 — VIO e navegação sem GPS
  • VIO (Visual-Inertial Odometry): localização com câmera + IMU
  • VINS-Mono, OpenVINS: implementações open-source
  • Navegação indoor sem GPS: desafios e soluções
  • Optical flow para navegação de baixa altitude
  • ROS 2 + PX4: integração para autonomia avançada
Mês 8 — IA para decisão de missão
  • Behavior trees para missões de drone (BehaviorTree.CPP)
  • Desvio de obstáculos: sensores de distância, câmera stereo
  • Detecção e seguimento de alvo (follow-me com visão)
  • Pouso de precisão com visão computacional
  • Projeto: drone seguindo alvo visual autonomamente na simulação

Fase 3 — Sistemas avançados e integração real (Meses 9–14)

Mês 9–10 — Swarm de drones
  • Coordenação multi-agente: algoritmos de formação
  • Comunicação entre drones: MAVLink2, WiFi mesh
  • Decolagem e aterrissagem sincronizada de múltiplos drones
  • Aplicação: cobertura colaborativa de área (busca e resgate)
Mês 11–12 — RL para controle de voo
  • Treinar controlador de voo com RL (Isaac Gym ou AirSim)
  • PPO para controle de multirotor: política de atitude e posição
  • Sim-to-real: adaptar política treinada para hardware real
  • Robustez a perturbações: vento, cargas variáveis
Mês 13–14 — Hardware real + projeto final
  • Montagem ou uso de drone F450 / F550 com Pixhawk
  • Companion computer: Raspberry Pi / NVIDIA Jetson Orin Nano
  • Pipeline completo: missão autônoma com detecção ativa em campo
  • Documentação, vídeo de demo, relatório de missão
Custo hardware mínimo: R$ 2.500–6.000 (drone + Pixhawk + câmera). NVIDIA Jetson Orin Nano (~R$ 1.500) para IA embarcada pesada.
Marco da trilha: missão autônoma completa demonstrada em vídeo. Pipeline de mapeamento funcionando. Habilitação ANAC obtida para operações comerciais.
