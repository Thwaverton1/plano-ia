⚙️ Trilha — Automação Inteligente com IA
Do zero ao engenheiro de automação com IA · ~16 meses · RPA, agentes, processos industriais e sistemas de controle com inteligência
Automação com IA é a área que mais cresce em todos os setores. Diferente do RPA tradicional (frágil), automação com IA usa visão, linguagem e raciocínio — e é muito mais robusta e valiosa.

Tipos de automação e onde cada um tem mercado

Automação de processos (RPA + IA)

  • RPA clássico: UiPath, Automation Anywhere — automação de cliques e formulários
  • RPA inteligente: visão computacional + OCR + LLM para processos complexos
  • Extração de dados de documentos: NF-e, contratos, laudos
  • Automação de atendimento: chatbots com RAG integrado a sistemas
  • Mercado: bancos, seguradoras, governo, varejo, logística

Agentes de IA (nova fronteira)

  • Agentes autônomos: LLM + ferramentas + memória + planejamento
  • Multi-agent systems: agentes especializados colaborando
  • Computer use: agentes controlando desktop/browser como humano
  • Automação de workflows: Zapier, Make, n8n com IA
  • Mercado: startups, consultorias, qualquer empresa com processos repetitivos

Automação industrial (IIoT)

  • SCADA / PLC com IA: detectar anomalias em processo industrial
  • Manutenção preditiva: sensores + modelos de ML para prever falhas
  • Qualidade automatizada: inspeção visual com CV em linha de produção
  • Mercado: indústria, mineração, petróleo, utilities

Fase 1 — Fundação (Meses 1–3)

Mês 1 — Automação com Python
  • Selenium / Playwright: automação de browser (web scraping, testes, RPA)
  • pyautogui: controle de mouse e teclado
  • Requests + BeautifulSoup: coleta de dados da web
  • Agendamento: schedule, APScheduler, cron
  • Projeto: bot que coleta dados, processa e envia relatório automático
Mês 2 — APIs e integração de sistemas
  • REST APIs: consumir, autenticar (OAuth2, API keys), tratar erros
  • Webhooks: receber eventos em tempo real
  • FastAPI: criar APIs para exposição de automações
  • Mensageria: Redis, RabbitMQ para filas de tarefas
  • Celery: execução assíncrona de tarefas pesadas
Mês 3 — OCR e extração de documentos
  • Tesseract OCR: extrair texto de imagens e PDFs
  • PaddleOCR: OCR moderno e mais preciso (open-source)
  • PyMuPDF, pdfplumber: extrair dados de PDFs estruturados
  • Extração com LLM: usar GPT/Claude para interpretar documentos
  • Projeto: pipeline de extração de dados de NF-e ou contratos em PDF

Fase 2 — Agentes de IA (Meses 4–8)

Mês 4 — LLMs como motor de automação
  • Function calling / Tool use: LLM decidindo qual ferramenta chamar
  • LangChain: chains, tools, memory, prompt templates
  • Structured outputs: forçar JSON estruturado do LLM
  • Projeto: agente que responde perguntas buscando em APIs externas
Mês 5 — LangGraph e agentes com estado
  • LangGraph: agentes com grafo de estados (muito mais robusto que chains)
  • ReAct pattern: raciocinar → agir → observar → repetir
  • Memória de curto e longo prazo em agentes
  • Human-in-the-loop: quando o agente pede confirmação humana
  • Projeto: agente de pesquisa que busca, sintetiza e escreve relatório
Mês 6 — Multi-agent systems
  • Frameworks: CrewAI, AutoGen, LangGraph multi-agent
  • Padrões: supervisor, hierarchical, collaborative
  • Agentes especializados: pesquisador, escritor, revisor, executor
  • Projeto: sistema multi-agente para análise de empresa (pesquisa + análise + relatório)
Mês 7–8 — Computer Use e automação visual
  • Agentes controlando desktop/browser como humano
  • Visão computacional para identificar elementos na tela
  • n8n / Make: workflows visuais com IA integrada
  • Integração com sistemas legados sem API (usando visão)
  • Projeto: agente que executa processo completo em sistema web

Fase 3 — Automação Industrial e Produção (Meses 9–14)

Mês 9–10 — IIoT e sensores industriais
  • MQTT: protocolo de mensageria para IoT industrial
  • Node-RED: programação visual para fluxos IoT
  • InfluxDB: banco de dados de séries temporais para sensores
  • Grafana: dashboard de monitoramento em tempo real
  • Projeto: coleta de dados de sensores → alertas automáticos → dashboard
Mês 11–12 — Manutenção preditiva com ML
  • Séries temporais: ARIMA, Prophet, LSTM para previsão de falha
  • Detecção de anomalias: Isolation Forest, autoencoders
  • Feature engineering para dados de vibração, temperatura, pressão
  • Deploy do modelo em pipeline real ou simulado
  • Projeto: sistema preditivo de falha em equipamento (motor, bomba)
Mês 13–14 — Sistema completo em produção
  • Arquitetura completa: sensores → ingestão → modelo → ação → alerta
  • Containers Docker + orquestração com docker-compose
  • Monitoramento do sistema em produção
  • Projeto final: solução de automação completa resolvendo problema real
Dica: contato com industrias locais em Goiás para implementar projeto real — vira case de portfólio poderoso
Marco da trilha: agente de IA completo deployado + pipeline industrial funcionando com dados reais ou simulados. Demonstração em vídeo de 5 minutos mostrando o processo antes e depois.
